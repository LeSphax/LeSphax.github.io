Reinforcement learning


Technologies : Python, Unity C#

I got interested in Reinforcement Learning in January 2018 when I heard about Unity ML agents. I got familiar with this framework and got it to work on extremely simple agents. For example, a catapult that should take a given orientation.
But I didn't understand actor critic algorithms so I didn't manage to train more complex agents.

To get a better understanding, I watched David Silver's course on Reinforcement Learning. Then I implemented the PPO algorithm by looking at the paper and the openAI baselines code. I made sure the performance was comparable to the baselines algorithm on the CartPole and the Breakout environments.

SlideBall


This game is multiplayer. If you just want to test it, you can open two windows (opening two tabs won't work) to have both client and host perspective.


Technologies : Unity C#

Tools : GIMP, Blender, Maya, Audacity

I really liked the mod Hover on Starcraft II and I tried to create a similar game by myself.

For the networking side, I started to use Photon Unity Network but I then switched to WebRTC to get a better latency. So I had to create several functionnalities like RPCs and distributing packets to the different entities. I am also hosting a signaling server to connect the players to each other.

I stopped working on this game on December 2017, in its current state it can be quite fun to play with 6 or 8 people.

Perspective Shift


Technologies : Unity C#

This is a puzzle game that progressively introduce new mechanics to the player that they must understand to progress further. At the moment there are only puzzles and no real structure to navigate them.

The original idea for the 2D game mechanics came from a friend, he also created the 3D models. Other than that, I built everything in this demo.

Implementing the undo feature while having animations between the different board states was quite interesting. As well as the logic for switching between the 2D and 3D modes.
I also spent a good amount of time designing the levels.

I stopped working on it in September 2016 and decided to focus on SlideBall instead.

FlyHigher


Technologies : Unity C#, XML
Tools : GIMP

As a group of 4 students, we made a game to teach aeronautics to children from 6 to 10 years old. We also produced a small paper containing recommandation for creating educational entertainment.

After learning how to use Unity, I created two of the 9 small games, the save management and translation modules and half of the game UI and navigation.

Designing, developing and testing the game took about 100 hours of work from each of us. We also organised and conducted user tests with primary school children.

Waves


Technologies : Unity C#

In this game, you follow a wave signal across several beautiful environments. The signal fades with time and you have to gather other waves to keep it strong. You can do this by changing the signal's amplitude and frequency.

I worked on this game with 4 artists, 1 sound designer and 2 developpers. I programmed everything related to the signal, its controls, the way it looks and how it changes the sound. I also programmed the waves that you collect and the "friend" waves that join you when the universe changes.

We spent a lot of time integrating making and integrating the art but not so much working on the gameplay, so the resulting game looks and sound nice, but it isn't really fun to play.
